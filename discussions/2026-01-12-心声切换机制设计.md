# 心声切换机制设计

日期: 2026-01-12
参与者: Human, Claude

## 背景

讨论如何实现多模型协作，探索更接近"内心对话"的机制。

---

## 关键发现

### DeepSeek R1 已有自我修正能力

通过 RL 训练，R1 自发涌现了：
- 自我反思
- 错误发现与修正（"aha moment"）
- 动态策略调整

这意味着单模型已经能"内部纠结"，但这不等于自我意识。

### 自我意识的两个条件

```
1. 连续存在（跨时间）
2. 意识到自己的存在（元认知）
```

当前 AI 两者都不满足。R1 的自我修正是推理能力，不是"我知道我在推理"。

---

## 核心设计决策

### AI 与人的底层逻辑不同

```
人类：独立大脑，独立记忆
AI：共享大脑（权重），不共享记忆（上下文）
    → 像神的分身
```

### 不是"合并"，是"切换"

```
合并逻辑（否决）：
A 说一段 + B 说一段 → 拼在一起
= 还是两个人

切换逻辑（采用）：
思考流中，某个时刻是 A 在想，某个时刻切到 B
= 一个人，但倾向在切换
= 不自知，自然发生
```

### 切换模式 = 性格

```
不是"有哪些声音"决定性格
而是"怎么在声音间切换"决定性格

性格 = 切换的概率分布
```

### 使用同模型 + 不同 prompt

```
不是 V3 + R1（两个不同能力的模型）
而是 V3 + V3（同模型，不同性格 prompt）

性格 prompt ≠ 角色扮演
是微妙的倾向暗示，不是身份设定
```

## 切换机制设计

### 架构

```
Input →
  thinking_stream:
    while not done:
      if 触发切换:
        current = switch(current)
      tokens = current.complete(context)
      context += tokens
  → 语言单元整理 → 输出
```

### 切换触发方式（待验证）

```
方案A：转折语义 - 检测"但是"、"不过"
方案B：随机/节奏 - 每 N tokens 有概率切换
方案C：自然停顿 - complete 到停顿点切换
方案D：学习 - 根据反馈调整切换概率
```

### 性格维度参考

```
风险敏感 ←→ 机会敏感
细节关注 ←→ 整体关注
逻辑优先 ←→ 直觉优先
快速结论 ←→ 保持开放
```

## 简化决定

### 砍掉

- ❌ 种群进化
- ❌ 遗传机制
- ❌ 多代追踪

### 保留

- ✓ 单个体观察迭代
- ✓ 多倾向（多 prompt）
- ✓ 切换机制
- ✓ 切换的学习

## 最小实验计划

```
第一步：实现切换机制
├── 2-3 个倾向 prompt
├── 简单切换规则（先随机或按节奏）
└── 输出思维流

第二步：观察
├── 思维流的质量
├── 切换是否自然
└── 与单模型对比

第三步：加入学习
├── 根据反馈调整切换概率
└── 观察模式是否稳定形成"性格"
```
